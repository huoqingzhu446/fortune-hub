## 1. 用户体系与鉴权

### 登录方式

| 方式 | 场景 | 后端接口 |
|---|---|---|
| 微信登录 | 小程序内 `wx.login()` 获取 code | `POST /auth/wechat-login` |
| 开发模拟登录 | 开发环境 `dev-` 前缀 code | 同上（mock 模式） |
| 手机号登录 | 短信验证码登录 | `POST /auth/phone-code` → `POST /auth/phone-login` |

### 登录流程

```
用户打开小程序
  → 微信登录: wx.login() → POST /auth/wechat-login { code, nickname, avatarUrl }
    → 服务端: code → 微信 jscode2session → openid → 创建/更新 UserEntity → 生成 session token
    → 返回 { token, user: { nickname, zodiac, vipStatus, ... } }
    → 前端存储 token + user 到 localStorage

手机号登录:
  → POST /auth/phone-code { phone } → 发送短信验证码
  → POST /auth/phone-login { phone, code } → 验证码校验 → 返回 token
```

### Session 存储

| Key | 内容 |
|---|---|
| `fortune-hub-auth-token` | JWT session token |
| `fortune-hub-auth-user` | 用户信息对象（昵称、头像、星座、VIP 状态） |
| `fortune-hub-auth-meta` | 登录方式、过期时间 |

### 用户数据模型

```
UserEntity
├── openid / unionid / phone  → 登录标识
├── nickname / avatarUrl      → 个人资料
├── birthday / birthTime      → 用于八字和星座计算
├── zodiac / baziSummary      → 系统计算产物
├── fiveElements              → 五行分布 JSON
├── vipStatus / vipExpiredAt  → 会员状态
└── lastLoginAt / lastLoginProvider
```

### ⚠️ 断点

- **无 token 刷新机制**：session 过期（30 天）后用户需重新登录，无静默刷新
- **用户资料不完整**：60% 的新用户只有 openid，缺少 birthday / birthTime，导致八字和星座个性化内容降级

## 2. 首页聚合

### 页面结构

```
┌─────────────────────────────┐
│  HomeHero                   │  标题、副标题、日期、农历
├─────────────────────────────┤
│  StatusIndexCard            │  综合气运指数（0-100）+ 状态标签
├─────────────────────────────┤
│  FortuneActionCard          │  "今日占卜" 入口
├─────────────────────────────┤
│  InsightGrid (6 cards)      │  情绪 | 心理 | 八字 | 星座 | 性格 | 探索
├─────────────────────────────┤
│  TodayAdviceCard            │  今日建议 + 行动引导

│  QuickToolStrip             │  快捷工具（记录心情、查看报告等）
├─────────────────────────────┤

└─────────────────────────────┘
```

### 综合指数计算逻辑（服务端 home.service.ts）

```
综合气运指数 = 基础分(60)
  + 情绪信号加权（近期心情好 +5~15，低落 -5~15）
  + 性格测评加权（有结果 +3~8）
  + 八字信号加权（有排盘 +2~5）
  + 情绪自检信号（steady +10，watch ±0，support -8，urgent -15）
  + 资料完整度加权（有生日 +3，有出生地 +2）
  最终 clamp(0, 100)
```

| `GET /bazi/professional/records/:id/detail` | 专业记录详情 |

### 计算引擎（`bazi-engine.ts`，646 行）

```
输入: 公历日期 + 时辰 + 性别
  → lunar-typescript 公历转农历
  → 年柱: 农历年 → 天干地支
  → 月柱: 节气月 → 天干地支
  → 日柱: 公历日 → 干支计算
  → 时柱: 时辰 → 天干地支
  → 五行计数: 木火土金水各多少个
  → 日主: 日柱天干 → 五行属性
  → 喜用神: 基于五行生克关系推算
输出: 四柱 + 五行分布 + 日主 + 建议
```

### 数据持久化

| 存储位置 | 内容 |
|---|---|
| 服务端 `UserRecordEntity` | 排盘结果（recordType: `bazi`） |
| 服务端 `UserEntity.fiveElements` | 最新五行分布 |
| 服务端 `UserEntity.baziSummary` | 八字摘要字符串 |

### ✅ 闭环状态

完整闭环：输入信息 → 排盘计算 → 结果展示 → 历史保存 → 可回看。

## 5. 周易占卜

### 页面结构（7 页完整流程）

```
┌──────────────────────┐
│ 1. 占卜首页 (539 行)  │  主题选择 + 状态同步
└────────┬─────────────┘
         ↓
┌──────────────────────┐
│ 2. 主题选择 (520 行)  │  话题轮播 + 问题输入（≤200字）+ 维度开关
└────────┬─────────────┘
         ↓
┌──────────────────────┐
│ 3. 起卦动画 (958 行)  │  方法选择（蓍草/抽签）+ 凝神动画 ⚠️ 无返回按钮
└────────┬─────────────┘
         ↓
┌──────────────────────┐
│ 4. 卦象结果 (1526 行) │  本卦 + 动爻 + 变卦 + 解读 + 操作
└────────┬─────────────┘
         ↓
    ┌────┴────┬────────────┐
    ↓         ↓            ↓
  分享海报   加收藏       重来
    ↓
┌──────────────────────┐
│ 5. 历史记录           │  过往占卜列表
└────────┬─────────────┘
         ↓
┌──────────────────────┐
│ 6. 复盘列表           │  服务端同步的复盘记录
└────────┬─────────────┘
         ↓
┌──────────────────────┐
│ 7. 分享海报           │  Canvas 客户端渲染 750×1334
└──────────────────────┘
```

### 核心问题：占卜计算纯客户端

起卦、卦象生成、解读**全部在客户端本地完成**（`services/divination-casting.ts` + `divination-content.ts`）。服务端只做复盘同步：

| 接口 | 用途 |
|---|---|
| `GET /divination/reviews` | 拉取已同步的复盘记录 |
| `POST /divination/reviews/sync` | 同步复盘到服务端 |

### 数据流

```
用户输入 → 客户端起卦算法 → 本地生成卦象+解读 → 本地 localStorage 保存
  → （可选）用户点"复盘" → POST /divination/reviews/sync → 服务端 DivinationReviewEntity
```

### ⚠️ 断点

| 问题 | 影响 |
|---|---|
| 起卦页无返回按钮 | 用户无法中途取消 |
| 纯客户端计算 | 换设备丢失所有历史；无法内容优化 |
| 无服务端起卦接口 | 无法做反作弊、频率限制 |
| 复盘同步非自动 | 用户需手动触发，大部分用户不会操作 |

## 6. 性格测评

### 页面结构（1 页，1062 行，4 屏状态机）

```
hub (总览) → list (题表列表) → quiz (答题) → result (结果)
```

### 各屏详情

**hub 屏**：展示 5 个测评模块（星座/八字/性格/情绪/幸运物），引导用户选择。

- `GET /assessments/personality/tests` 拉取题库列表

- 逐题展示，选择选项后自动下一题
- 每题有维度标签（如"行动感"、"稳定感"、"感受力"）

**result 屏**：
- 总分 + 维度分数
- 性格画像（如"轻快执行型"）：标题、摘要、优势、建议
- 分享海报预览
- 操作：生成海报 / 复制文案 / 查看完整版 / 再测一次

| 接口 | 用途 |
|---|---|
| `GET /assessments/personality/tests` | 题库列表 |
| `GET /assessments/personality/tests/:code` | 题目详情 |
| `POST /assessments/personality/tests/:code/submit` | 提交答案 |
| `GET /assessments/personality/history` | 历史记录 |

### ✅ 闭环状态

完整闭环：选择题库 → 答题 → 评分 → 结果展示 → 保存历史 → 可回看。分享海报入口直达 poster 生成页。

## 7. 情绪自检

### 页面结构（1 页，1513 行，3 屏）

```
list (题表) → quiz (答题) → result (结果)
```

与性格测评流程类似，关键差异：

### 安全优先设计

- **安全警告条**：页面顶部固定显示"如果已经持续失眠、明显失控，或出现伤害自己的想法，请立即联系现实中的家人朋友、医院、急救或当地心理危机干预热线。"
- **免责声明**：结果页显示"本结果仅用于日常自我观察，不构成医学诊断或治疗建议"
- **放松建议**：结果页包含 3 条放松步骤

### 评分系统

- 情绪自检使用**阈值判定**（非维度评分）
- 每题 0-3 分，总分映射到 4 个风险等级：

| 分数范围 | 等级 | 标签颜色 | 建议强度 |
|---|---|---|---|
| 0-6 | steady | 绿色 | 保持现状 |
| 7-12 | watch | 黄色 | 关注变化 |
| 13-18 | support | 橙色 | 建议寻求支持 |
| 19+ | urgent | 红色 | 强烈建议专业咨询 |

### API 接口

| 接口 | 用途 |
|---|---|
| `GET /assessments/emotion/tests` | 题库列表 |
| `GET /assessments/emotion/tests/:code` | 题目详情 |
| `POST /assessments/emotion/tests/:code/submit` | 提交答案 |

### ✅ 闭环状态

完整闭环：选择量表 → 答题 → 风险评级 → 结果+建议 → 保存历史。

## 10. 分享海报

### 两套渲染系统

| 系统 | 技术 | 调用方 | 输出尺寸 |
|---|---|---|---|
| 服务端 | sharp + SVG | 星座/八字/情绪/性格/幸运签 | 941×1672 |
| 客户端 | Canvas (微信) | 占卜/八字客户端版 | 750×1334 |

### 海报生成流程（服务端）

```
前端 POST /posters/jobs { sourceType, recordId, size }
  → posters.service.ts: resolvePosterSource() → 组装 PosterSource 数据
  → posterRendererService.resolvePosterLayout() → 确定尺寸
  → posterRendererService.renderPoster() → SVG 模板 → sharp → PNG
  → 上传到文件服务 → 返回 fileUrl
  → 前端轮询 GET /posters/jobs/:jobId → 拿到结果 → 展示
```

### 海报类型

| sourceType | SVG 模板函数 | 特色 |
|---|---|---|
| `zodiac_today` | `renderZodiacTemplatePoster` | 星座 PNG 底图 + SVG 文字叠加 |
| `bazi` | `buildBaziPosterSvg` | 四柱命盘 + 五行装饰 |
| `emotion` | `buildEmotionAssessmentPosterSvg` | 雷达图 + 建议网格 |
| `today_index` | `buildRichPosterSvg` | 综合指数 + 指标块 |
| `lucky_sign` | `buildSharePosterSvg` | 幸运签分享卡 |
| 测评结果 | `buildRichPosterSvg` | 分数 + 维度 |

### ⚠️ 断点
