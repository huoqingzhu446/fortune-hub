## 5. 小程序审核材料清单

- 小程序名称、简介、头像、服务类目与实际功能一致。
- 隐私保护指引已在微信后台配置，并覆盖 openid、手机号、生日、出生时间、性别、头像昵称、反馈附件、订阅消息等数据项。
- 用户协议、隐私政策、免责声明在小程序内可访问。
- 情绪自检页面明确“非医疗诊断”，高风险结果提供求助建议。
- 支付相关页面、商品名称、价格、权益、退款/客服说明清楚一致。
- 订阅消息模板内容与触发场景一致，不诱导订阅。
- 分享海报不包含违规承诺、医疗暗示、夸大收益或无法访问的小程序码。
- 正式域名、文件服务域名、业务域名均已配置到微信后台合法域名。
- 体验版二维码、测试账号、审核说明准备好。

## 6. 发布前验收路径

1. 新建或确认 release 分支，工作区保持干净。
2. 配置 staging 环境，使用 `DB_SYNCHRONIZE=false` 和 `DB_RUN_MIGRATIONS=true` 部署。
3. 在 staging 写入正式候选内容、会员套餐、题库、配置中心数据。
4. 真机跑通微信登录、资料完善、首页、探索、星座、八字、占卜、性格测评、情绪自检、记录、报告、海报、会员支付、设置、隐私、反馈、订阅消息。
5. 跑构建与测试：`pnpm test:api`、`pnpm --filter @fortune-hub/api test:e2e`、`pnpm build:api`、`pnpm build:admin`、`pnpm type-check:mobile`、`pnpm build:mobile`。
6. 使用微信开发者工具导入 `apps/mobile/dist/build/mp-weixin`，开启 URL 校验后预览。
7. 上传体验版，完成产品/运营/法务最后验收。
8. 提交审核，记录版本号、git commit、构建时间、环境变量版本和回滚方案。
