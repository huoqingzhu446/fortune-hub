# 小程序首页 UI 剩余改造计划

> 更新日期：2026-05-11  
> 适用范围：`apps/mobile/src/pages/index/index.vue`、首页相关组件、`services/api/src/home/home.service.ts`、后台配置中心  
> 当前目标：在已完成“今日状态中枢”首轮 UI 改造后，继续把首页做成可配置、可运营、可解释、可验证的产品入口。

---

## 1. 当前基线

### 1.1 已完成方向

首页首轮 UI 已从“卡片入口聚合”调整为“今日状态中枢”：

1. 首屏结构调整为：今日头部、状态主卡、今日一步、状态洞察、轻量占卜、快捷工具。
2. 状态主卡开始承载状态指数、依据、可信度、免责声明和主次行动。
3. 新增“今日一步”模块，用于根据用户阶段给出唯一优先行动。
4. 占卜和快捷工具被降级为次级入口，减少首屏注意力竞争。
5. H5 类型检查与构建已通过，移动端宽度检查无横向滚动。

### 1.2 仍需补齐的问题

本轮已针对 1.2 的问题完成第一批落地，当前状态如下：

1. [x] 首页模块、顺序、文案、CTA 接入后台配置约定：`home/index_layout`。
2. [x] “今日一步”的状态规则沉到 `GET /home/index`，前端只消费 `todayAction`。
3. [x] 心绪记录入口已合并到“今日一步”的 `record_mood` 状态，并复用记录成功后刷新首页的现有链路。
4. [x] 后台配置中心新增“首页设置模板”和轻量可视化编辑区，支持模块、适用人群、灰度、快捷工具配置。
5. [~] 首页已补曝光与点击埋点方法，当前先落到本地 analytics 入口；后续还需接入真实日志/数仓统计。

仍需在后续阶段继续增强：移动端配置预览、真实埋点上报、真机多尺寸验收。

---

## 2. 剩余改造总目标

1. 配置化：后台可控制首页模块开关、顺序、快捷工具、主行动文案和灰度。
2. 个性化：未登录、资料未完成、状态依据不足、压力偏高、VIP 用户看到不同首页行动。
3. 可解释：所有状态指数必须展示依据、可信度和非诊断声明。
4. 可验证：每个主要入口具备曝光、点击和转化埋点。
5. 可维护：前端组件只负责渲染，业务规则优先来自 `GET /home/index` 或配置。

### Phase 2：今日一步规则服务端化

优先级：P0  
目标：把“用户现在最该做什么”的判断从前端计算迁移到后端聚合结果。

任务清单：

- [x] 在 `GET /home/index` 返回 `todayAction` 字段。
- [x] 后端统一判断用户状态：未登录、资料未完成、依据不足、压力偏高、状态可推进、VIP 权益推荐。
- [x] 前端 `TodayActionCard` 只消费接口字段，不再重复写复杂判断。
- [x] 为高压力或低情绪场景提供更安全的文案，不出现诊断式表达。
- [x] 为每个行动提供 `actionCode`，便于埋点统计。

建议返回结构：

```json
{
  "todayAction": {
    "actionCode": "complete_profile",
    "badge": "资料待完善",
    "title": "补齐生日与出生信息",
    "summary": "资料完整后，首页会把状态观察、八字节奏和长期画像放在同一条线上看。",
    "primaryText": "完善资料",
    "primaryRoute": "/pages/profile/index",
    "secondaryText": "先记录心情",
    "secondaryRoute": "/pages/journal/index"
  }
}
```

验收标准：

- [x] 前端删除大部分今日一步判断逻辑后，页面表现不退化。
- [x] 未登录、资料未完成、依据不足、压力偏高、状态可推进五类状态都能稳定返回不同行动。
- [x] 所有行动文案遵守“观察参考，不做诊断”的表达边界。

## 5. 风险与注意事项

1. 不要直接删除 `DailyPulseCard` 现有逻辑，先改入口关系，避免影响已记录隐藏逻辑。
2. 首页配置必须有默认回退，不能因为后台配置错误导致首页白屏。
3. 路由配置必须校验是否存在于 `pages.json`，避免运营误配死链。
4. 今日一步的高压力场景必须保持克制，不应引导用户做重大决策。
5. 底部导航为固定层，所有首屏模块都要按真实视口验证遮挡。

---

## 6. 完成定义

剩余改造完成后，应满足以下条件：

- [x] 首页结构、快捷工具和主行动可通过后台配置调整。
- [x] 首页主行动由后端统一生成，前端只负责渲染。
- [x] 用户不同阶段看到不同的首页重点。
- [x] 心绪记录入口不重复、不抢首屏、不打扰已记录用户。
- [ ] 首页在 375px、390px、430px 常见宽度下无横向滚动和关键遮挡。
- [~] 首页核心点击行为可被统计。当前已有前端事件入口，后续接真实上报与统计面板。
- [~] 所有心理、情绪、运势相关表达保持参考和陪伴边界。当前首页主行动已收敛，后续补全局词表。
