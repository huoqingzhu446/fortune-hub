## 1. 项目目标

`Fortune Hub` 定位为一个围绕“运势内容 + 轻测评 + 用户成长 + 后台运营”展开的多端产品，核心能力包括：

- 运势内容：星座、八字、幸运签、幸运指数、幸运物推荐
- 测评能力：性格测评、情绪自检
- 用户成长：历史记录、分享海报、VIP 会员、广告解锁
- 后台运营：题库管理、内容配置、报告模板、广告与会员配置

当前仓库已经从“多端联调骨架”推进到了“首版业务原型”阶段，已经具备：

- 小程序端 / H5 首版功能页
- 管理端登录、题库管理、内容中心、商业化配置首版
- API 服务首版业务接口与基础商业化闭环
- MySQL / Redis / Docker / Nginx 联调与部署链路

但它仍不是完整可上线产品。当前更准确的定位是：

- 可演示：是
- 可继续运营化扩展：是
- 已形成完整商业化闭环：否
- 已具备完整后台：否

```text
↓

```

## 3. 技术架构

### 3.1 目标架构

```text
微信小程序（客户端）
├─ 页面层
├─ 逻辑交互层
└─ 数据缓存层

云函数 / 后端接口服务
├─ 业务接口服务
├─ 核心算法服务
└─ 内容管理服务

数据持久层
├─ 数据库
├─ 文件存储
└─ 配置中心
```

### 3.2 当前仓库实际架构

当前代码仓库采用 `pnpm workspace` 多应用结构：

- `apps/mobile`
  小程序端 / H5，基于 `uni-app + Vue 3 + Vite + TypeScript + Pinia`
- `apps/admin`
  管理端，基于 `Vue 3 + Vite + Element Plus`
- `services/api`
  API 服务，基于 `NestJS + TypeORM + MySQL + Redis`
- `deploy/nginx`
  H5、管理端、API 的 Nginx 反向代理配置

当前仍是单体 API 承载业务，并未拆分独立算法服务、配置中心或内容服务。

```text
├─ 用户ID
├─ 测评类型
└─ 结果数据

share 海报表
├─ 分享类型
├─ 生成内容
└─ 生成时间

ad 广告配置表
├─ 激励视频开关
└─ 广告位配置
```

### 4.2 当前实际数据层

当前代码已经接入以下实体 / 表：

- `assessment_sessions`
  早期联调用示例表，仍存在，但不是主业务表
- `users`
  用户资料、星座、简易八字摘要、五行、VIP 状态等
- `records`
  性格测评、情绪自检、八字解读等历史结果
- `fortune_contents`
  幸运签及部分内容化文案
- `assessment_questions`
  性格测评 / 情绪自检题目
- `assessment_test_configs`
  测试集基础信息、结果画像、情绪阈值、分享海报配置
- `assessment_test_groups`
  题库后台运营分类
- `orders`
  会员订单与支付状态
- `shares`
  分享海报生成记录
- `ad_configs`
  广告位与激励配置
- `membership_products`
  会员套餐与权益配置

当前结论：

- `users`、`records`、`fortune_contents` 已承载首版业务
- 题库已经不再完全写死在服务端常量里，`assessment_questions`、`assessment_test_configs`、`assessment_test_groups` 已投入使用
- `orders`、`shares`、`ad_configs`、`membership_products` 已补齐，`configs`、`lucky_items`、`report_templates` 仍待落地
- 当前仍依赖 TypeORM `synchronize`，尚未切换到 migration

## 5. 模块拆解

### 5.1 小程序端 / H5

#### 首页

当前实现：

- 已有真实首页页面与 `GET /api/v1/home/index`
- 已展示今日幸运指数、年度幸运指数、幸运签、功能入口
- 已支持跳转到星座、八字、性格、情绪、幸运物、个人中心、历史、设置与会员入口
- 已提供自定义底部导航和 H5 / 小程序双运行模式
- 首页幸运签已可进入独立详情页
- 首页会根据当前登录态与资料完善度返回个性化闭环提示与快捷入口

缺口：

- 还没有原生 `tabBar` 级别的全局导航
- 首页个性化推荐仍偏轻量，主要围绕登录、资料与幸运体系闭环
- 首页推荐与后台内容配置尚未打通

#### 星座模块

当前实现：

- 已有独立星座页
- 已有完整首版接口组：
  - `GET /api/v1/zodiac/daily`

  - `GET /api/v1/zodiac/yearly`

  - `GET /api/v1/zodiac/knowledge`
- 已支持 12 星座切换、日 / 周 / 年视图、配对和知识卡

缺口：

- 内容仍以内置规则和文案为主

- 星座结果分享海报仍未单独接入

## 7. 代码核对结论

基于当前仓库代码，项目状态可以归纳为：

- 已完成的是“多端工程底座 + 首版业务原型 + 报告海报与基础商业化闭环”
- 部分完成的是“正式登录、幸运体系深化、内容后台化、后台生命周期”
- 尚未完成的是“完整模板体系、独立幸运物库、正式支付与 migration”

这意味着：

- 项目已经能支撑演示、继续开发和一轮小范围联调
- 项目还不适合直接作为完整商业化产品上线
