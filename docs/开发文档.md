## 1. 项目目标

`Fortune Hub` 定位为一个围绕“运势内容 + 轻测评 + 用户成长 + 后台运营”展开的多端产品，核心能力包括：

- 运势内容：星座、八字、幸运签、幸运指数、幸运物推荐
- 测评能力：性格测评、情绪自检
- 用户成长：历史记录、分享海报、VIP 会员、广告解锁
- 后台运营：题库管理、内容配置、报告模板、广告与会员配置

当前仓库已经从“多端联调骨架”推进到了“首版业务原型”阶段，已经具备：

- 小程序端 / H5 首版功能页
- 管理端题库管理首版
- API 服务首版业务接口
- MySQL / Redis / Docker / Nginx 联调与部署链路

但它仍不是完整可上线产品。当前更准确的定位是：

- 可演示：是
- 可继续运营化扩展：是
- 已形成完整商业化闭环：否
- 已具备完整后台：否

### 2.2 用户核心流程

```text
用户打开小程序
↓
微信授权登录 → 自动获取 openid
↓
完善生日信息 → 自动计算星座 / 简易八字
↓
进入功能页：
  星座运势 ←→ 八字解读 ←→ 性格测评 ←→ 情绪自检 ←→ 幸运体系
↓
生成结果报告
↓
免费查看精简版
或
看广告 / 开通 VIP → 解锁完整版
↓
生成分享海报
↓
保存 / 转发朋友圈 / 好友
```

## 3. 技术架构

### 3.1 目标架构

```text
微信小程序（客户端）
├─ 页面层
├─ 逻辑交互层
└─ 数据缓存层

云函数 / 后端接口服务
├─ 业务接口服务
├─ 核心算法服务
└─ 内容管理服务

数据持久层
├─ 数据库
├─ 文件存储
└─ 配置中心
```

### 3.2 当前仓库实际架构

当前代码仓库采用 `pnpm workspace` 多应用结构：

- `apps/mobile`
  小程序端 / H5，基于 `uni-app + Vue 3 + Vite + TypeScript + Pinia`
- `apps/admin`
  管理端，基于 `Vue 3 + Vite + Element Plus`
- `services/api`
  API 服务，基于 `NestJS + TypeORM + MySQL + Redis`
- `deploy/nginx`
  H5、管理端、API 的 Nginx 反向代理配置

当前仍是单体 API 承载业务，并未拆分独立算法服务、配置中心或内容服务。

## 4. 数据模型设计

### 4.1 目标数据表

```text
user 用户表
├─ openid
├─ 生日 / 性别
├─ 星座 / 五行
└─ VIP 状态

config 配置表
├─ 星座运势
├─ 八字文案
├─ 测评题库
└─ 幸运物库

order 订单表
├─ 订单号
├─ 用户ID
├─ 支付状态
└─ 开通时长

record 记录表
├─ 用户ID
├─ 测评类型
└─ 结果数据

share 海报表
├─ 分享类型
├─ 生成内容
└─ 生成时间

ad 广告配置表
├─ 激励视频开关
└─ 广告位配置
```

### 4.2 当前实际数据层

当前代码已经接入以下实体 / 表：

- `assessment_sessions`
  早期联调用示例表，仍存在，但不是主业务表
- `users`
  用户资料、星座、简易八字摘要、五行、VIP 状态等
- `records`
  性格测评、情绪自检、八字解读等历史结果
- `fortune_contents`
  幸运签及部分内容化文案
- `assessment_questions`
  性格测评 / 情绪自检题目
- `assessment_test_configs`
  测试集基础信息、结果画像、情绪阈值、分享海报配置
- `assessment_test_groups`
  题库后台运营分类

当前结论：

- `users`、`records`、`fortune_contents` 已承载首版业务
- 题库已经不再完全写死在服务端常量里，`assessment_questions`、`assessment_test_configs`、`assessment_test_groups` 已投入使用
- `configs`、`orders`、`shares`、`ad_configs`、`lucky_items` 等目标表仍未落地
- 当前仍依赖 TypeORM `synchronize`，尚未切换到 migration

## 5. 模块拆解

### 5.1 小程序端 / H5

#### 首页

当前实现：

- 已有真实首页页面与 `GET /api/v1/home/index`
- 已展示今日幸运指数、年度幸运指数、幸运签、功能入口
- 已支持跳转到星座、八字、性格、情绪、幸运物、个人中心
- 已提供自定义底部导航和 H5 / 小程序双运行模式
- 首页幸运签已可进入独立详情页

缺口：

- 还没有原生 `tabBar` 级别的全局导航
- 首页个性化推荐仍偏轻量
- 首页推荐与后台内容配置尚未打通

#### 星座模块

当前实现：

- 已有独立星座页
- 已有完整首版接口组：
  - `GET /api/v1/zodiac/daily`
  - `GET /api/v1/zodiac/weekly`
  - `GET /api/v1/zodiac/yearly`
  - `GET /api/v1/zodiac/compatibility`
  - `GET /api/v1/zodiac/knowledge`
- 已支持 12 星座切换、日 / 周 / 年视图、配对和知识卡

缺口：

- 内容仍以内置规则和文案为主
- 没有后台发布流
- 还没有星座结果分享海报

#### 八字模块

当前实现：

- 已有独立八字页
- 已有 `POST /api/v1/bazi/analyze`
- 已有 `GET /api/v1/bazi/history`
- 已支持出生信息录入、简化四柱、五行倾向、建议和合规提示
- 已支持登录后写入历史

缺口：

- 当前仍是“简化排盘体验版”
- 未接入节气换月、真太阳时和更完整规则
- 未实现海报生成与后台内容配置

#### 性格测评

当前实现：

- 已有测评列表、答题页、结果页、历史记录
- 已有完整接口组：
  - `GET /api/v1/assessments/personality/tests`
  - `GET /api/v1/assessments/personality/tests/:code`
  - `POST /api/v1/assessments/personality/tests/:code/submit`
  - `GET /api/v1/assessments/personality/history`
- 题目已落到 `assessment_questions`
- 测试集基础配置、结果画像、分享海报配置已落到 `assessment_test_configs`
- 管理端已可编辑题目、结果画像和海报模板配置
- 结果页已能展示 `sharePoster` 预览信息并复制海报文案

缺口：

- 还没有真正生成图片海报
- 结果分层、会员解锁、广告解锁未实现
- 仍缺管理员登录和更完整的题库生命周期管理

#### 情绪自检

当前实现：

- 已有两套轻量问卷与结果流
- 已有完整接口组：
  - `GET /api/v1/assessments/emotion/tests`
  - `GET /api/v1/assessments/emotion/tests/:code`
  - `POST /api/v1/assessments/emotion/tests/:code/submit`
  - `GET /api/v1/assessments/emotion/history`
- 题目已落表
- 免责声明、放松建议、阈值分段和海报配置已可通过后台题库配置驱动
- 已支持登录后保存历史

缺口：

- 量表仍是轻量体验版
- 还没有独立合规页、热线配置和危机分流配置
- 还没有真正海报生成服务

#### 幸运体系模块

当前实现：

- 已有独立幸运物页和幸运签详情页
- 已有 `GET /api/v1/lucky/today`
- 已有 `GET /api/v1/lucky/signs/:bizCode`
- 已有 `POST /api/v1/lucky/wallpaper/generate`
- `GET /api/v1/lucky/today` 已返回今日 / 年度幸运指数、幸运签、幸运物推荐、壁纸方向
- 推荐已能结合用户星座、五行和最近八字 / 性格 / 情绪结果做首版联动
- 已有壁纸生成页，可生成并预览 SVG 壁纸，H5 下可直接下载

缺口：

- 推荐规则仍在服务端内置，未形成独立规则表
- 还没有幸运物内容库后台和年度详情页
- 壁纸当前为 SVG 首版生成，尚未接图片服务、海报模板和文件落库

#### 个人中心

当前实现：

- 已有个人中心页
- 已支持 `GET /api/v1/me`、`PUT /api/v1/me/profile`
- 已支持 H5 开发环境登录与小程序 `uni.login` 登录入口
- 已支持登录时尝试同步微信头像和昵称
- 已显示 OpenID、VIP 状态、星座、简易八字摘要、五行分布
- 已有统一历史记录页，对应 `GET /api/v1/records`
- 已有设置中心、隐私说明、意见反馈、关于我们页面首版

缺口：

- 正式微信授权链路仍需真机环境验证与失败态补齐
- 还没有 VIP 中心和权益说明
- 推送开关仍是本地偏好设置，尚未接系统推送或服务端配置

  - `/me/profile`

## 7. 代码核对结论

基于当前仓库代码，项目状态可以归纳为：

- 已完成的是“多端工程底座 + 首版业务原型 + 题库配置首版”
- 部分完成的是“正式登录、幸运体系深化、分享传播、后台运营化”
- 尚未完成的是“商业化闭环、完整后台、完整内容体系和完整数据模型”

这意味着：

- 项目已经能支撑演示、继续开发和一轮小范围联调
- 项目还不适合直接作为完整商业化产品上线
