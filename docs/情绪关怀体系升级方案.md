# 情绪关怀体系升级方案

> 更新时间：2025-07-17
> 核心目标：帮助用户了解当前情绪状态，并进行有方向性的状态引导，让产品从"测评工具"进化为"情绪陪伴系统"。

---

## 一、产品定位反思

当前产品的情绪相关功能：

| 功能 | 做的事 | 缺失 |
|---|---|---|
| 情绪自检 | 用标准化量表筛查低落和焦虑 | 做完就结束，无后续跟进 |
| 情绪日记 | 记录每天心情 | 只有记录，没有洞察 |
| 冥想记录 | 记录练习前后状态 | 记录工具，不是引导工具 |
| 占卜 | 用户提问 → 起卦 → 解读 | 未承担情绪角色，仪式感浪费了 |

**核心问题**：每个功能都是"单次使用后即结束"的工具，没有形成 **"感知 → 理解 → 干预 → 反馈 → 成长"** 的闭环。

**关键洞察**：占卜本质上就是一种情绪缓解工具——用户在焦虑/迷茫时来问卦，起卦的仪式感本身就是一次冥想，卦象解读提供了新的思考角度。但目前产品没有把这个价值显性化。

## 三、具体功能设计

### 3.1 每日情绪签到（Daily Pulse Check）

**位置**：首页顶部，Hero 区域下方

**流程**：
```
用户打开小程序
  → 首页显示 "今天感觉怎么样？"
  → 5 个情绪图标：😊 不错 / 😐 一般 / 😔 低落 / 😰 焦虑 / 😤 烦躁
  → 点击后展开二级选项（可选）：
    - 选 😔 → "是因为什么？" → 工作 / 人际 / 健康 / 不确定
    - 选 😰 → "紧张程度？" → 轻度/中度/重度
  → 可选补充一句话（≤100字）
  → 完成 → 显示一句共情回应 + 个性化建议
```

**数据存储**（扩展现有 `MoodRecordEntity`）：
```typescript
DailyPulseRecord {
  date: string           // 日期 YYYY-MM-DD
  mood: 'happy' | 'neutral' | 'low' | 'anxious' | 'irritable'
  intensity: 1-5         // 强度
  category?: string      // 归因分类
  note?: string          // 用户留言（≤100字）
  responseMood?: string  // 干预后的情绪（回填）
}
```

**共情回应库**（根据情绪类型）：
| 情绪 | 回应 |
|---|---|
| 😊 不错 | "真好！今天的你充满能量。继续保持这样的节奏。" |
| 😐 一般 | "平平淡淡也是真。不如试试下面的呼吸练习，给自己加一点新鲜感？" |
| 😔 低落 | "我听到了。低落不是软弱，是身体在提醒你需要休息。需要我陪你做点什么吗？" |
| 😰 焦虑 | "焦虑是你对在乎的事情足够认真。我们一起用 2 分钟把注意力带回当下。" |
| 😤 烦躁 | "感觉什么都压着？先别急着解决，做一次深呼吸再回来。" |

**后端接口**：
- `POST /me/pulse` — 提交今日签到
- `GET /me/pulse?from=&to=` — 拉取签到历史
- `GET /me/pulse/streak` — 连续签到天数

### 3.3 引导式呼吸练习（Guided Breathing）

**位置**：首页快捷工具 → 新入口"快速平静"，或情绪低落/焦虑后自动推荐

**流程**：
```
┌─────────────────────────────────────┐
│         呼吸练习（4-7-8 法）          │
│                                     │
│         ⬤                          │
│        /  \      ← 动画圆圈          │
│       /    \       吸气 4 秒 放大    │
│      /      \      屏息 7 秒 保持    │
│      \      /      呼气 8 秒 缩小    │
│       \    /                        │
│        \  /                         │
│         ⬤                          │
│                                     │
│     吸气... 4秒                       │
│     ━━━━━━━━━━━━━━━━━━              │
│     已完成 3/5 轮                     │
│                                     │
│     [暂停] [跳过] [结束练习]           │
└─────────────────────────────────────┘
```

**模式**：
| 模式 | 节奏 | 适合场景 |
|---|---|---|
| 4-7-8 放松法 | 吸4-屏7-呼8 | 睡前、焦虑 |
| 盒式呼吸 | 吸4-屏4-呼4-屏4 | 专注、考试前 |
| 5-5 平静法 | 吸5-呼5 | 日常放松 |
| 2-4 快速法 | 吸2-呼4 | 急性焦虑 |

**后端接口**：
- `GET /wellness/breathing/modes` — 呼吸模式列表
- `POST /record/breathing` — 保存练习记录（模式、轮数、前后情绪分）

### 3.5 情绪引导引擎（Recommendation Engine）

**核心**：基于用户当前情绪状态，推荐最合适的干预方式

```
输入：用户今日情绪签到 + 近7日情绪趋势 + 历史干预效果
  ↓
规则引擎：
  if 连续3天低落 → 推荐"5分钟正念冥想" + "情绪日记"
  if 急性焦虑 → 推荐"2-4 快速呼吸法"
  if 情绪平稳 → 推荐"占卜"探索新视角
  if 烦躁易怒 → 推荐"身体扫描冥想" + "情绪日记"
  if 刚做完冥想且情绪提升 → "你做得很好！保持这个习惯"
  if 很久没签到 → "有几天没来看你了，今天感觉怎么样？"
  ↓
输出：首页展示个性化行动卡片
```

**后端**：
- `GET /wellness/recommendations` — 获取今日推荐
- 引擎数据来源：`MoodRecordEntity` + `DailyPulseRecord` + `MeditationRecordEntity` + `UserRecordEntity` + `DivinationReviewEntity`

## 四、实现优先级

### Phase 1 — 情感感知层（1-2 周）

| 功能 | 后端 | 前端 | 优先级原因 |
|---|---|---|---|
| 每日情绪签到 | `POST/GET /me/pulse` + `DailyPulseRecord` entity | 首页签到组件 | 所有后续功能的数据基础 |
| 签到连续天数 | `GET /me/pulse/streak` | 签到后展示天数 | 养成习惯的激励机制 |

### Phase 2 — 可视化理解层（2-3 周）

| 功能 | 后端 | 前端 |
|---|---|---|
| 情绪报告 | `GET /me/emotion-report` | 个人中心新增"情绪报告"入口 + 热力图/折线图 |
| 情绪洞察 | `GET /me/emotion-insights` | 报告页底部 AI 洞察卡片 |

### Phase 3 — 干预引导层（2-3 周）

| 功能 | 后端 | 前端 |
|---|---|---|
| 引导呼吸练习 | `GET /wellness/breathing/modes` + `POST /record/breathing` | 新页面：呼吸练习 + CSS 动画 |
| 占卜情绪化改造 | `POST /divination/cast` + `POST /divination/feedback` + entity 扩展 | 起卦前情绪选择 + 出卦后反馈组件 |
| 情绪引导引擎 | `GET /wellness/recommendations` + 规则引擎 | 首页个性化行动卡片 |

### Phase 4 — 成长闭环层（1-2 周）

| 功能 | 后端 | 前端 |
|---|---|---|
| 紧急支持卡片 | 热线数据配置化 | 首页/自检结果/设置三处替换 |
| 每周情绪报告推送 | 微信订阅消息（已有 PushSubscriptionEntity） | 模板消息 |

## 五、改造前后对比

```
改造前：
  用户打开 → 做测评 → 看结果 → 关闭
  用户焦虑 → 来占卜 → 看卦象 → 关闭
  用户记录心情 → 保存 → 关闭
  （每个功能是孤岛，用完即走）

改造后：
  用户打开 → 情绪签到 → 首页推荐今日引导
    → 焦虑 → 呼吸练习 → "感觉好些了吗？" → "继续保持！"
    → 低落 → 呼吸练习 + 占卜 → "试试换个角度看看？"
    → 迷茫 → 占卜(情绪锚定) → 出卦 → "这个视角有帮助吗？"
    → 平稳 → "看看你这周的情绪变化" → 情绪报告
  每周日 → 订阅消息推送 → "这周你焦虑 2 天，比上周少了 1 天 🎉"
  （形成感知→理解→干预→反馈→成长的完整闭环）
```

## 七、关键指标

上线后需追踪的核心指标：

| 指标 | 目标 | 衡量方式 |
|---|---|---|
| 情绪签到日活率 | >30% 日活用户完成签到 | `每日签到数 / DAU` |
| 签到连续天数中位数 | >4 天 | `MEDIAN(streak)` |
| 干预完成率 | >60% 推荐被点击并完成 | `完成干预数 / 推荐曝光数` |
| 干预有效率 | >50% 情绪提升 | `postMood 好于 preMood 的比例` |
| 占卜情绪改善率 | >40% | `占卜后情绪优于占卜前的比例` |
| 周活跃留存 | >25% | `本周活跃 ∩ 下周活跃 / 本周活跃` |

## 八、一句话总结

> 当前产品帮用户"看见"情绪（测评 + 日记），但没帮用户"处理"情绪（引导 + 反馈）。这次升级的核心思路是：**把占卜、呼吸、感恩、冥想串联成一条情绪引导链路，让用户从"我知道我焦虑"走向"我知道怎么让自己平静下来"。**
