# Fortune Hub 数据库设计文档

更新时间：2026-04-22  
数据库类型：MySQL 8.x

## 1. 文档目标

本文件用于沉淀 `Fortune Hub` 的目标数据库结构，并核对当前仓库已经实现的数据层能力。

当前数据层结论：

- 已完成：MySQL 接入、TypeORM 接入、`users`、`records`、`fortune_contents`、`assessment_sessions` 实体
- 部分完成：用户资料、结果历史、幸运签等内容的首版持久化
- 未完成：配置、订单、分享、广告、题库配置、幸运物内容库等核心表

## 2. 设计原则

目标设计原则：

- 优先使用 `snake_case`
- 所有业务主表建议使用 `bigint unsigned` 自增主键
- 金额字段使用 `decimal(10,2)`
- 状态字段优先使用 `varchar` 或 `tinyint`
- 需要扩展的结构化内容优先使用独立表；弹性较高的结果数据使用 `json`

当前代码实际情况：

- TypeORM 默认创建的是 camelCase 字段，而不是 `snake_case`
- 当前尚未接入 migration，开发环境以 `synchronize` 自动同步表结构为主
- 因此“目标设计稿”和“当前真实表结构”存在命名差异，后续若要统一需要额外迁移

说明：

- 当前尚未承载分享、海报、商业化分层

## 6. 表关系建议

### 当前实际关系

```text
users
└─ 1:N records

fortune_contents
└─ 暂时独立使用，供首页幸运签等内容读取

assessment_sessions
└─ 早期示例表，未参与主业务闭环
```

### 目标扩展关系

```text
users
├─ 1:N orders
├─ 1:N records
└─ 1:N shares

records
└─ 1:N shares

assessment_questions
└─ 为 personality / emotion 提供题库来源

fortune_contents
└─ 为 zodiac / lucky / bazi 提供内容来源

lucky_items
└─ 为幸运物推荐提供内容来源
```
