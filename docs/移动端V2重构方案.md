# 今日气运移动端 V2 重构方案

## 1. 目标

本轮重构目标不是局部优化，而是在保留现有 monorepo、`uni-app + Vue3 + Pinia + NestJS` 基础设施的前提下，将当前偏“功能验证型工具集合”的移动端，重构为符合 PRD 与设计稿的「疗愈陪伴型小程序」。

重构重点分为三条主线：

1. 产品结构重构：重建首页、探索、记录、我的四个一级页面职责。
2. 视觉系统重构：建立奶白浅色、轻国风、轻玄学、毛玻璃卡片的统一主题体系。
3. 数据与接口重构：让前端页面结构与后端聚合接口一一对应，避免继续使用开发验证期的临时字段。

---

## 2. 当前判断

### 2.1 现状问题

当前移动端存在以下核心错位：

1. 底部导航错位
当前为 `首页 / 测评 / 幸运物 / 我的`，目标应为 `首页 / 探索 / 记录 / 我的`。

2. 首页职责错位
当前首页更像“聚合入口页”，目标首页应是“今日状态中心”，围绕综合气运指数、四宫格状态、今日建议展开。

3. 设计语言错位
当前页面风格偏蓝灰、英文辅助文案、验证型 UI；目标风格为奶白浅色、国风留白、轻玄学氛围、月相山水、毛玻璃卡片。

4. 数据结构错位
当前 `services/api/src/home/home.service.ts` 输出的字段偏开发验证语义，不适合直接承载 V2 视觉与交互。

### 2.2 重构原则

1. 不换技术栈，先重建结构。
2. 不直接覆盖旧页，采用 V2 增量替换。
3. 先搭主题系统，再改首页，再扩到探索/记录/我的。
4. 所有页面统一从主题层取色，不允许页面内写死主色。

## 4. 主题系统方案

### 4.1 核心结论

整体风格支持两种设置方式：

1. 自动模式：默认跟随用户“今日幸运色 / 今日主题”
2. 手动模式：用户在设置页手动固定主题

主题优先级统一为：

`手动主题 > 今日幸运色主题 > 系统默认主题`

### 4.2 主题模式定义

建议新增：

```ts
export type ThemeMode = 'auto' | 'manual';

export type ThemeKey =
  | 'mist_blue'
  | 'sakura_pink'
  | 'turquoise'
  | 'lavender'
  | 'milk_apricot'
  | 'champagne_gold'
  | 'moon_silver'
  | 'peach_orange'
  | 'rose_dust'
  | 'mint_cyan'
  | 'sea_salt'
  | 'jade_white'
  | 'amber_honey'
  | 'cloud_gray_purple';
```

### 4.3 前端偏好字段

现有 `apps/mobile/src/services/preferences.ts` 中的 `AppSettings` 建议扩展为：

```ts
export interface AppSettings {
  dailyReminderEnabled: boolean;
  luckyPushEnabled: boolean;
  quietModeEnabled: boolean;
  saveHistoryCardsEnabled: boolean;
  themeMode: ThemeMode;
  manualThemeKey: ThemeKey | '';
}
```

默认值建议：

```ts
const DEFAULT_SETTINGS: AppSettings = {
  dailyReminderEnabled: true,
  luckyPushEnabled: true,
  quietModeEnabled: false,
  saveHistoryCardsEnabled: true,
  themeMode: 'auto',
  manualThemeKey: '',
};
```

### 4.4 运行时主题解析结果

页面不要自行判断来源，统一通过解析器产出 `effectiveTheme`：

```ts
export interface ResolvedThemeState {
  mode: ThemeMode;
  manualThemeKey: ThemeKey | '';
  dailyThemeKey: ThemeKey | '';
  effectiveThemeKey: ThemeKey;
  source: 'manual' | 'daily' | 'fallback';
}
```

推荐解析逻辑：

```ts
function resolveEffectiveTheme(input: {
  themeMode: ThemeMode;
  manualThemeKey?: ThemeKey | '';
  dailyThemeKey?: ThemeKey | '';
  fallbackThemeKey?: ThemeKey;
}): ResolvedThemeState
```

规则：

1. `themeMode === 'manual'` 且 `manualThemeKey` 有值时，使用手动主题。
2. 否则若存在 `dailyThemeKey`，使用今日主题。
3. 否则回退到默认主题，建议 `mist_blue` 或 `milk_apricot`。

### 4.5 主题 token 结构

建议新增：

```txt
apps/mobile/src/theme/
  tokens.ts
  themes.ts
  useTheme.ts
  glass.scss
  page-shell.scss
```

`themes.ts` 中每个主题至少包含：

```ts
export interface ThemePalette {
  key: ThemeKey;
  name: string;
  primary: string;
  soft: string;
  accent: string;
  pageBg: string;
  cardBg: string;
  textPrimary: string;
  textSecondary: string;
  tagBg: string;
  border: string;
  glow: string;
}
```

### 4.6 页面取色原则

所有一级页面统一从 `effectiveThemeKey` 派生变量：

- 页面标题
- 大分数
- 进度条
- 当前 Tab
- 标签背景
- CTA 边框或浅填充
- 背景插画光晕

禁止：

- 在页面内直接写死 `#5f8dff`
- 不同页面自定义“主色体系”
- 某些卡片绕开主题系统单独上色

## 5. 设置页交互方案

### 5.1 设置页定位

设置页不再只是推送开关页，而是“偏好中心”，承载：

1. 整体风格
2. 提醒偏好
3. 隐私与反馈入口

### 5.2 推荐模块结构

#### 模块一：整体风格

字段与交互：

1. 跟随今日幸运色
2. 手动选择主题
3. 当前生效主题展示
4. 手动主题预览卡列表

推荐文案：

- 模块标题：`整体风格`
- 模块描述：`默认跟随今日幸运色，也可以固定成你更喜欢的疗愈主题。`

模式项文案：

- `跟随今日幸运色`
- `手动选择主题`

状态提示文案：

- 自动模式：`今日主题：雾蓝色`
- 手动模式：`已固定为：奶杏色，不随每日变化`

#### 模块二：提醒偏好

保留现有字段：

1. 每日幸运提醒
2. 幸运物推荐提醒
3. 安静模式
4. 保留历史卡片

#### 模块三：更多入口

保留现有入口：

1. 隐私说明
2. 意见反馈
3. 关于我们

### 5.3 主题选择交互

#### 自动模式

- 默认选中
- 显示“今日主题”提示
- 手动主题网格保留展示，但视觉弱化
- 点击手动主题卡时自动切换为手动模式并应用

#### 手动模式

- 高亮当前手动主题卡
- 立即预览当前主题
- 保存后全局生效

### 5.4 主题卡建议

每个主题卡展示：

1. 主题名
2. 2-3 个渐变色块
3. 1 行氛围描述

例如：

- 雾蓝：安静、清澈
- 奶杏：温暖、稳定
- 薰衣草：梦幻、冥想
- 香槟金：高级、幸运
- 月光银：冷静、澄明
- 豆沙玫瑰：情绪、陪伴

### 5.5 设置页视觉建议

设置页本身也使用当前 `effectiveThemeKey` 渲染，但要保持轻盈，避免成为高饱和色板页：

- 顶部 Hero 用淡山水和小范围光晕
- 主题模式使用胶囊切换
- 主题卡用两列瀑布或两列网格
- 当前生效主题显示为一条通栏玻璃条

## 7. 接口与数据模型建议

### 7.1 首页接口

建议新增或重构为：

`GET /api/home/today`

返回重点：

```json
{
  "date": "2026-04-24",
  "title": "今日气运",
  "subTitle": "身心和谐 · 顺势而为",
  "fortune": {
    "score": 86,
    "level": "good",
    "summary": "状态平稳，适合自我疗愈与整理内心",
    "advice": "保持平和与专注，温柔地对待自己",
    "luckyColorName": "雾蓝色",
    "luckyColorValue": "#6F91B7",
    "luckyNumber": 7,
    "suitableActions": ["静心", "整理", "呼吸"],
    "themeKey": "mist_blue"
  }
}
```

前端仅取 `fortune.themeKey` 作为自动模式下的 `dailyThemeKey`。

### 7.2 用户偏好接口

建议新增：

`GET /api/user/preferences`

```json
{
  "themeMode": "auto",
  "manualThemeKey": ""
}
```

`PUT /api/user/preferences`

```json
{
  "themeMode": "manual",
  "manualThemeKey": "milk_apricot"
}
```

### 7.3 持久化策略

#### 未登录用户

- 使用 `uni.setStorageSync` 保存在本地

#### 已登录用户

- 本地立即更新
- 后台异步同步用户偏好
- 登录后服务端偏好覆盖本地临时值，或进行一次合并策略

推荐规则：

1. 未登录先用本地偏好
2. 登录后拉用户偏好
3. 若服务端无记录，则以上一次本地设置为准写回服务端

### 7.4 服务端存储建议

两种可选方案：

#### 方案 A：挂在 `user` 表

新增字段：

- `themeMode`
- `manualThemeKey`

适合当前快速迭代。

#### 方案 B：新增 `user_preferences`

字段建议：

- `userId`
- `themeMode`
- `manualThemeKey`
- `dailyReminderEnabled`
- `luckyPushEnabled`
- `quietModeEnabled`
- `saveHistoryCardsEnabled`

更利于后续扩展通知、冥想时长、音频偏好等设置。

建议：V2 第一阶段可先用方案 A，后续再拆独立表。

## 8. Phase 1 实施清单

第一阶段目标：先完成 V2 的“壳层基础设施”，不急着一次性改完所有业务页面。

### 8.1 工程基础

1. 新建 `theme/` 目录
2. 建立 `ThemeKey`、`ThemeMode`、`ThemePalette` 类型
3. 建立 `themes.ts` 主题映射
4. 建立 `resolveEffectiveTheme` 解析函数

### 8.2 设置与偏好

1. 扩展 `AppSettings`
2. 新增主题偏好默认值
3. 封装 `getAppSettings / saveAppSettings` 的主题支持
4. 新增 `useThemePreference`

### 8.3 主题注入

1. 页面统一接入 `effectiveThemeKey`
2. 页面根节点挂载主题样式变量
3. 抽离统一玻璃卡样式
4. 抽离统一标题/正文/辅助文字层级

### 8.4 V2 导航

1. 重构 `AppTabBar`
2. 调整为 `首页 / 探索 / 记录 / 我的`
3. 保留旧业务页为二级页

### 8.5 设置页改版

1. 增加“整体风格”模块
2. 增加自动/手动切换
3. 增加主题预览卡
4. 增加“当前生效主题”提示条

### 8.6 首页 V2 骨架准备

1. 新建首页标题区
2. 接入 `dailyThemeKey`
3. 预留 `FortuneScoreCard`
4. 预留四宫格与建议卡组件结构

### 8.7 验收标准

第一阶段完成后应满足：

1. 设置页可以切换自动/手动主题
2. 手动模式切换后，全局页面主色立即变化
3. 自动模式下，首页可跟随当天 `themeKey`
4. 今日无主题数据时，自动回退到默认主题
5. 所有一级页面不再写死主色
