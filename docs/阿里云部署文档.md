# Fortune Hub 阿里云部署文档

更新时间：2026-04-22

## 1. 适用范围

这份文档用于把 `fortune-hub` 部署到阿里云 ECS 服务器上，默认采用：

- `Docker + Docker Compose`
- `Nginx` 作为统一入口
- `MySQL + Redis + API + Admin + Mobile H5` 同机部署

当前这套部署方案适合：

- 移动端 H5 页面
- 管理端 Web 页面
- API 服务
- MySQL / Redis 持久化运行

需要注意：

- 当前 `docker-compose.yml` 部署的是 `mobile-h5 + admin + api + mysql + redis + nginx`
- 微信小程序端不是通过 Nginx 直接静态托管上线的
- 小程序仍然需要你在本地构建后，通过微信开发者工具上传

当前路由约定：

- `/`：移动端 H5
- `/admin/`：管理端
- `/api/`：API 入口

## 2. 服务器准备

建议的阿里云 ECS 基线：

- 系统：Ubuntu 22.04 / Alibaba Cloud Linux 3 / CentOS 7+
- CPU / 内存：至少 `2C4G`
- 磁盘：至少 `40GB`
- 已绑定公网 IP
- 已完成域名解析到这台 ECS

阿里云安全组建议放行：

- `22`：SSH
- `80`：HTTP
- `443`：HTTPS

不建议直接对公网放行：

- `3306`
- `6379`

当前仓库已经把 MySQL 和 Redis 默认改成只绑定 `127.0.0.1`，更适合服务器使用。


## 5. 准备服务器环境变量

复制阿里云专用环境模板：

```bash
cp .env.aliyun.example .env.aliyun
```

你至少需要修改这些字段：

- `APP_DOMAIN`
  你的正式域名，例如 `fortune.example.com`
- `ENABLE_HTTPS`
  有证书就填 `true`，没有证书可以先填 `false`
- `SSL_CERT_SOURCE`
  证书源文件路径
- `SSL_KEY_SOURCE`
  私钥源文件路径
- `MYSQL_PASSWORD`
- `MYSQL_ROOT_PASSWORD`
- `CORS_ORIGIN`
  一般写成 `https://你的域名`
- `FILE_SERVICE_BASE_URL`
  如果你有独立文件服务，这里填正式地址

推荐配置示例：

```env
APP_DOMAIN=fortune.example.com
ENABLE_HTTPS=true
SSL_CERT_SOURCE=/data/certs/fullchain.pem
SSL_KEY_SOURCE=/data/certs/privkey.pem
DEPLOY_BRANCH=main

MYSQL_BIND_HOST=127.0.0.1
MYSQL_HOST=mysql
MYSQL_PORT=3306
MYSQL_DATABASE=fortune_hub
MYSQL_USER=fortune
MYSQL_PASSWORD=change-this-password
MYSQL_ROOT_PASSWORD=change-this-root-password

REDIS_BIND_HOST=127.0.0.1
REDIS_HOST=redis
REDIS_PORT=6379

API_PORT=3001
NGINX_BIND_HOST=0.0.0.0
NGINX_HTTP_PORT=80
NGINX_HTTPS_PORT=443
CORS_ORIGIN=https://fortune.example.com

FILE_SERVICE_BASE_URL=https://file.example.com/api
```

如果你当前先按服务器公网 IP `8.152.214.57` 直接部署，还没有正式域名和证书，可以先用这组思路：

```env
APP_DOMAIN=8.152.214.57
ENABLE_HTTPS=false
NGINX_HTTP_PORT=80
CORS_ORIGIN=http://8.152.214.57
FILE_SERVICE_BASE_URL=http://8.152.214.57:3000/api
```

也就是说：

- 先用 `HTTP` 跑通移动端 H5、管理端和 API
- 后面有正式域名和证书后，再切到 `HTTPS`
- 当前仓库根目录里我已经按这个 IP 生成了一份 `.env.aliyun`

## 6. 准备证书

如果你要走 HTTPS，有两种方式：

### 方式 A：把证书源路径写进 `.env.aliyun`

例如：

```env
SSL_CERT_SOURCE=/data/certs/fullchain.pem
SSL_KEY_SOURCE=/data/certs/privkey.pem
```

部署脚本会自动把它们复制到：

- `deploy/nginx/ssl/fullchain.pem`
- `deploy/nginx/ssl/privkey.pem`

### 方式 B：手动放到项目目录

```bash
mkdir -p deploy/nginx/ssl
cp /your/fullchain.pem deploy/nginx/ssl/fullchain.pem
cp /your/privkey.pem deploy/nginx/ssl/privkey.pem
```

如果你暂时没有证书，可以先在 `.env.aliyun` 里设置：

```env
ENABLE_HTTPS=false
```

这样会先按纯 HTTP 部署。

需要额外注意：

- 纯 IP + HTTP 方案适合先把移动端 H5、管理端和 API 跑起来
- 如果后面要接入微信小程序正式请求域名，通常还是需要备案域名和 HTTPS，而不是长期使用公网 IP
- 当前纯 IP 部署下：
  `http://8.152.214.57/` 是移动端 H5
  `http://8.152.214.57/admin/` 是管理端

## 7. 一键部署

仓库已经提供阿里云部署脚本：

- `scripts/deploy-aliyun.sh`

首次部署：

```bash
bash scripts/deploy-aliyun.sh deploy
```

如果你是服务器上长期更新代码后再重新部署：

```bash
bash scripts/deploy-aliyun.sh pull-and-deploy
```

这个脚本会做这些事：

- 读取 `.env.aliyun`
- 检查并复制 SSL 证书
- 按 HTTP / HTTPS 模式渲染 Nginx 配置
- 执行 `docker compose up -d --build`

## 8. 常用运维命令

查看容器状态：

```bash
bash scripts/deploy-aliyun.sh status
```

查看日志：

```bash
bash scripts/deploy-aliyun.sh logs
```

重启容器：

```bash
bash scripts/deploy-aliyun.sh restart
```

停止整套服务：

```bash
bash scripts/deploy-aliyun.sh down
```

只重新渲染 Nginx 配置：

```bash
bash scripts/deploy-aliyun.sh render-nginx
```

## 10. 建议的发布流程

建议每次发布都按下面的顺序走：

1. 本地开发并验证
2. 提交代码并 push 到 GitHub
3. 在服务器上执行 `git pull`
4. 运行 `bash scripts/deploy-aliyun.sh pull-and-deploy`
5. 检查 `status` 和 `logs`

如果你后续发布了正式 release tag，也可以在服务器上切到对应版本再部署：

```bash
git fetch --tags
git checkout <实际发布的tag>
bash scripts/deploy-aliyun.sh deploy
```

## 11. 当前方案的已知限制

- 当前部署方案默认托管 `mobile-h5 + admin + api`
- 微信小程序端仍需要单独构建并上传
- 星座、八字、测评内容仍以内置数据为主，不是完全后台配置化
- 当前没有单独的生产级数据库备份脚本

## 12. 后续建议
