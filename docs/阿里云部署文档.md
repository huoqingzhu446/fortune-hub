# Fortune Hub 阿里云部署文档

更新时间：2026-04-23

## 1. 适用范围

这份文档用于把 `fortune-hub` 部署到阿里云 ECS 服务器上，默认采用：

- `Docker + Docker Compose`
- `Nginx` 作为统一入口
- `MySQL + Redis + API + Admin + Mobile H5` 同机部署

当前这套部署方案适合：

- 移动端 H5 页面
- 管理端 Web 页面
- API 服务
- MySQL / Redis 持久化运行

需要注意：

- 当前 `docker-compose.yml` 部署的是 `mobile-h5 + admin + api + mysql + redis + nginx`
- 微信小程序端不是通过 Nginx 直接静态托管上线的
- 小程序仍然需要你在本地构建后，通过微信开发者工具上传

当前路由约定：

- `/`：移动端 H5
- `/admin/`：管理端
- `/api/`：API 入口

## 2. 服务器准备

建议的阿里云 ECS 基线：

- 系统：Ubuntu 22.04 / Alibaba Cloud Linux 3 / CentOS 7+
- CPU / 内存：至少 `2C4G`
- 磁盘：至少 `40GB`
- 已绑定公网 IP
- 已完成域名解析到这台 ECS

阿里云安全组建议放行：

- `22`：SSH
- `80`：HTTP
- `443`：HTTPS

不建议直接对公网放行：

- `3306`
- `6379`

当前仓库已经把 MySQL 和 Redis 默认改成只绑定 `127.0.0.1`，更适合服务器使用。

## 3. 安装依赖

先在服务器安装基础依赖：

```bash
sudo apt update
sudo apt install -y git curl ca-certificates
```

安装 Docker 与 Docker Compose：

```bash
curl -fsSL https://get.docker.com | sh
sudo usermod -aG docker "$USER"
newgrp docker
docker --version
docker compose version
```

## 5. 准备服务器环境变量

复制阿里云专用环境模板：

```bash
cp .env.aliyun.example .env.aliyun
```

你至少需要修改这些字段：

- `APP_DOMAIN`
  你的正式域名，例如 `fortune.example.com`
- `ENABLE_HTTPS`
  有证书就填 `true`，没有证书可以先填 `false`
- `SSL_CERT_SOURCE`
  证书源文件路径
- `SSL_KEY_SOURCE`
  私钥源文件路径
- `MYSQL_PASSWORD`
- `MYSQL_ROOT_PASSWORD`
- `ADMIN_USERNAME`
- `ADMIN_PASSWORD`
- `CORS_ORIGIN`
  一般写成 `https://你的域名`
- `FILE_SERVICE_BASE_URL`
  如果你有独立文件服务，这里填正式地址

推荐配置示例：

```env
APP_DOMAIN=fortune.example.com
ENABLE_HTTPS=true
SSL_CERT_SOURCE=/data/certs/fullchain.pem
SSL_KEY_SOURCE=/data/certs/privkey.pem
DEPLOY_BRANCH=main

MYSQL_BIND_HOST=127.0.0.1
MYSQL_HOST=mysql
MYSQL_PORT=3306
MYSQL_DATABASE=fortune_hub
MYSQL_USER=fortune
MYSQL_PASSWORD=use-a-long-random-mysql-password
MYSQL_ROOT_PASSWORD=use-a-long-random-mysql-root-password

ADMIN_USERNAME=ops-admin
ADMIN_PASSWORD=use-a-long-random-admin-password

REDIS_BIND_HOST=127.0.0.1
REDIS_HOST=redis
REDIS_PORT=6379

API_PORT=3001
DB_SYNCHRONIZE=false
DB_RUN_MIGRATIONS=true
NGINX_BIND_HOST=0.0.0.0
NGINX_HTTP_PORT=80
NGINX_HTTPS_PORT=443
CORS_ORIGIN=https://fortune.example.com

FILE_SERVICE_BASE_URL=https://fortune.example.com/file-api/api
PUBLIC_API_BASE_URL=https://fortune.example.com/api/v1
PAYMENT_MODE=disabled
SMS_PROVIDER=aliyun
SMS_MOCK_ENABLED=false
WECHAT_APP_ID=wx-your-mini-program-appid
WECHAT_APP_SECRET=use-real-wechat-secret
WECHAT_MINI_PROGRAM_ENV_VERSION=release
POSTER_MINI_PROGRAM_CODE_ENABLED=true
POSTER_MINI_PROGRAM_CODE_REQUIRED=false
POSTER_WXACODE_STRICT_PATH=true
POSTER_WXACODE_ALLOWED_PAGES=pages/index/index,pages/zodiac/index,pages/lucky/sign/index,pages/divination/index/index,pages/report/index
```

数据库结构发布建议：

- 生产环境保持 `DB_SYNCHRONIZE=false`
- 建议打开 `DB_RUN_MIGRATIONS=true`
- 部署脚本会拒绝 `DB_SYNCHRONIZE=true`、默认弱密码、生产 mock 短信和生产 mock 支付
- 当前版本已包含首个内容运营基础迁移，会补齐 `lucky_items`、`configs`、`report_templates` 和若干生命周期字段
- `DB_RUN_MIGRATIONS=true` 只会执行尚未运行过的迁移，后续保留开启也可以
- Docker Compose 已为 MySQL、Redis、API 配置 healthcheck，API 会等待数据库和 Redis healthy 后启动
- API 镜像、管理端镜像和移动端 H5 镜像都从 monorepo 根目录构建，并使用 `pnpm-lock.yaml` 的 `--frozen-lockfile` 安装依赖

如果你当前按正式域名 `www.yuanlian.xin` 和 HTTPS 部署，可以使用这组配置：

```env
APP_DOMAIN=www.yuanlian.xin
ENABLE_HTTPS=true
NGINX_HTTP_PORT=80
CORS_ORIGIN=https://www.yuanlian.xin
FILE_SERVICE_BASE_URL=https://www.yuanlian.xin/file-api/api
PUBLIC_API_BASE_URL=https://www.yuanlian.xin/api/v1
```

也就是说：

- 使用 `HTTPS` 对外提供移动端 H5、管理端和 API
- 域名证书放到 `deploy/nginx/ssl/` 后，由 Nginx 统一入口转发
- 可以按这组值生成或更新仓库根目录下的 `.env.aliyun`

## 6. 准备证书

如果你要走 HTTPS，有两种方式：

### 方式 A：把证书源路径写进 `.env.aliyun`

例如：

```env
SSL_CERT_SOURCE=/data/certs/fullchain.pem
SSL_KEY_SOURCE=/data/certs/privkey.pem
```

部署脚本会自动把它们复制到：

- `deploy/nginx/ssl/fullchain.pem`
- `deploy/nginx/ssl/privkey.pem`

### 方式 B：手动放到项目目录

```bash
mkdir -p deploy/nginx/ssl
cp /your/fullchain.pem deploy/nginx/ssl/fullchain.pem
cp /your/privkey.pem deploy/nginx/ssl/privkey.pem
```

如果你暂时没有证书，可以先在 `.env.aliyun` 里设置：

```env
ENABLE_HTTPS=false
```

这样会按 HTTP 临时模式部署，方便服务器内测。微信小程序正式版必须使用 HTTPS，因此提交审核前需要重新设置 `ENABLE_HTTPS=true` 并配置有效证书。

需要额外注意：

- 微信小程序正式请求域名通常需要备案域名和 HTTPS
- 当前域名部署下：
  `https://www.yuanlian.xin/` 是移动端 H5
  `https://www.yuanlian.xin/admin/` 是管理端

## 7. 一键部署

仓库已经提供阿里云部署脚本：

- `scripts/deploy-aliyun.sh`

首次部署：

```bash
bash scripts/deploy-aliyun.sh deploy
```

如果你是服务器上长期更新代码后再重新部署：

```bash
bash scripts/deploy-aliyun.sh pull-and-deploy
```

这个脚本会做这些事：

- 读取 `.env.aliyun`
- 检查并复制 SSL 证书
- 按 HTTP / HTTPS 模式渲染 Nginx 配置
- 执行 `docker compose up -d --build`

部署前建议先确认：

- `git status` 是干净的，尤其不要保留手改后的 `deploy/nginx/conf.d/default.conf`
- `.env.aliyun` 已配置 `DB_SYNCHRONIZE=false`
- `.env.aliyun` 已配置 `DB_RUN_MIGRATIONS=true`
- `.env.aliyun` 已配置正式 `ADMIN_USERNAME`、`ADMIN_PASSWORD`、`MYSQL_PASSWORD`、`WECHAT_APP_ID`、`WECHAT_APP_SECRET` 和 HTTPS 的 `FILE_SERVICE_BASE_URL`
- 如果海报必须携带小程序码，将 `POSTER_MINI_PROGRAM_CODE_REQUIRED=true`，并确认 `POSTER_WXACODE_ALLOWED_PAGES` 覆盖所有扫码落地页

## 8. 常用运维命令

查看容器状态：

```bash
bash scripts/deploy-aliyun.sh status
```

查看日志：

```bash
bash scripts/deploy-aliyun.sh logs
```

重启容器：

```bash
bash scripts/deploy-aliyun.sh restart
```

停止整套服务：

```bash
bash scripts/deploy-aliyun.sh down
```

只重新渲染 Nginx 配置：

```bash
bash scripts/deploy-aliyun.sh render-nginx
```

## 9. 部署完成后的检查

检查容器：

```bash
docker compose --env-file .env.aliyun ps
```

这里应能看到 MySQL、Redis、API 的健康状态。若 API 一直不是 healthy，优先看 API 日志中的生产配置校验错误。

检查健康接口：

```bash
curl https://www.yuanlian.xin/api/v1/health
```

也可以直接运行生产健康检查脚本，一次性验证 API、文件服务、移动端 H5 和管理端的 HTTPS 访问与证书校验：

```bash
pnpm health:prod
```

默认检查 `https://www.yuanlian.xin`。如需检查其他域名，可覆盖：

```bash
APP_DOMAIN=fortune.example.com pnpm health:prod
```

检查首页接口：

```bash
curl https://www.yuanlian.xin/api/v1/home/index
```

检查发布态数据：

```bash
curl -H "Authorization: Bearer <admin-session-token>" \
  https://www.yuanlian.xin/api/v1/admin/ops/release-readiness
```

该接口会返回题库、题目、内容、幸运物、报告模板、会员商品和合规配置的 `pass/warn/fail` 状态。正式提交审核前，核心业务项不应存在 `fail`。

如果管理端已部署成功，浏览器访问：

```text
https://www.yuanlian.xin/admin/
```

如果移动端 H5 已部署成功，浏览器访问：

```text
https://www.yuanlian.xin/
```

## 10. 建议的发布流程

建议每次发布都按下面的顺序走：

1. 本地开发并验证
2. 提交代码并 push 到 GitHub
3. 在服务器上执行 `git pull`
4. 运行 `bash scripts/deploy-aliyun.sh pull-and-deploy`
5. 检查 `status` 和 `logs`

如果你后续发布了正式 release tag，也可以在服务器上切到对应版本再部署：

```bash
git fetch --tags
git checkout <实际发布的tag>
bash scripts/deploy-aliyun.sh deploy
```

## 11. 当前方案的已知限制

- 当前部署方案默认托管 `mobile-h5 + admin + api`
- 微信小程序端仍需要单独构建并上传
- 星座、八字、测评内容仍以内置数据为主，不是完全后台配置化
- 当前没有单独的生产级数据库备份脚本

## 12. 后续建议

如果你准备把这套服务长期跑在阿里云上，建议继续补：

- `systemd` 守护的备份任务
- MySQL 定时备份脚本
- 日志轮转
- GitHub Actions 自动部署到阿里云
- 阿里云 SLB / WAF / 监控告警
